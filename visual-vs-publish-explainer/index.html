Skill map for this repo

Visual Explainer makes the page. Publish Explainer ships it.

These two skills form a simple production line: one turns complex material into a polished standalone HTML explanation; the other copies that finished HTML into the public explainers repo, updates the catalog, commits, pushes, and returns a shareable URL.

Created with

InputVisual Explainer starts from ideas, code, plans, diffs, data, or diagrams. Publish Explainer starts from an existing standalone .html file.
OutputVisual Explainer outputs local HTML in ~/.agent/diagrams/. Publish Explainer outputs a GitHub Pages URL plus repo changes.
BoundaryVisual Explainer is creative/rendering work. Publish Explainer is deployment/release work.
Maker / renderer

visual-explainer

Generates self-contained HTML visual explanations for systems, code changes, plans, data, and technical concepts.

  • 1Chooses a representation: Mermaid, CSS cards, tables, timelines, dashboards, or slide decks.
  • 2Designs the page: embedded CSS/JS, typography, palette, responsive layout, and visual hierarchy.
  • 3Produces an artifact: a complete standalone HTML document, normally opened locally.
handoff
HTML file
Shipper / publisher

publish-explainer

Publishes a standalone HTML explainer into the dedicated explainers GitHub repo as /<slug>/index.html.

  • 1Requires source HTML: stops and asks if the file path is missing.
  • 2Runs the publisher: copies the file, regenerates the catalog, commits, pushes, and checks Pages.
  • 3Returns delivery info: local path, public URL, Pages status, and commit/push status.

How they work together

create → verify → publish

02 / VISUAL

03 / VISUAL

Check layout, overflow, visual hierarchy, fonts, and browser behavior.

04 / PUBLISH

Copy the HTML into ~/prj/active/explainers/<slug>/index.html and rebuild the catalog.

05 / PUBLISH

Release

Commit, push, ensure GitHub Pages, then report the public URL.

Rules of thumb

routing

Use visual-explainer first when there is no page yet.

If the user asks for a diagram, visual summary, comparison, or HTML explainer, create the standalone page locally before thinking about publishing.

Use publish-explainer only after HTML exists.

It is intentionally narrow: verify a source .html, copy it into the explainers repo, update the catalog, commit, push, and return the URL.
